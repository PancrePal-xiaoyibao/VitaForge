# OODA Orient 阶段检查清单

> 用途: OODA 循环 Orient 阶段的标准化检查流程
> 版本: v1.0
> 更新: 2026-05-19

---

## 使用方法

在每次 OODA Orient 阶段，复制此清单并逐项检查：

```
## Orient 检查清单 — Step [XX]

### 1. 快速文献检索 (L1)

- [ ] 提取关键发现中的生物学概念
- [ ] 使用 PubMed MCP 检索关键词
- [ ] 使用 OpenAlex MCP 补充检索
- [ ] 记录检索结果 (PMID/DOI 或"未找到")

### 2. 结果评估

- [ ] 结果与文献一致？
  - [ ] 是 → 标记"文献验证通过"
  - [ ] 部分一致 → 标记"部分验证"
  - [ ] 矛盾 → 触发深度调研
  - [ ] 无直接文献 → 触发深度调研

### 3. 文献深度判定

- [ ] 是否直接支撑核心论点？ → L3 全文精读
- [ ] 是否涉及方法学细节？ → L2 关键段落
- [ ] 是否验证已有知识？ → L1 摘要级
- [ ] 默认 → L1 摘要级

### 4. 深度调研判定

- [ ] 是否需要系统性综述？ → 调用 /deep-research
- [ ] 是否需要方法学比较？ → 调用 /deep-research
- [ ] 是否需要跨领域支持？ → 调用 /deep-research
- [ ] 研究者是否明确要求？ → 调用 /deep-research

### 5. WORKLOG 记录

- [ ] 检索关键词已记录
- [ ] 找到文献已记录 (PMID/DOI)
- [ ] 验证结论已记录
- [ ] 解释依据已记录

### 6. README 更新

- [ ] 参考文献列表已添加
- [ ] 文献验证记录表已添加

## 快速参考

### 文献调研深度标准 (L1/L2/L3)

| 级别 | 处理方式 | 适用场景 |
|------|----------|----------|
| L1: 摘要级 | 读摘要 + 结论 | 验证一致性、背景补充、方法确认 |
| L2: 关键段落 | 读摘要 + 方法 + 结果 | 方法学细节、关键发现验证 |
| L3: 全文精读 | 下载全文 + 精读 | 核心文献、争议结论、方法复现 |

### deep-research 调用格式

```
/deep-research "[研究主题]"
目标: [具体调研目标]
背景: [当前结果和需要验证的内容]
预期产出: [文献列表/综述/方法比较]
```

### WORKLOG 文献验证字段格式

```markdown
- **文献验证**:
  - 检索关键词: [PubMed/OpenAlex 检索词]
  - 找到文献: [PMID/DOI] 或 "未找到直接文献"
  - 验证结论: [一致/部分一致/矛盾/新颖发现]
  - 解释依据: [文献内容摘要，1-2 句话]
```
