# 实验记录 (Worklog)

**项目**: {project-name}
**创建日期**: {YYYY-MM-DD}
**格式说明**: 每个 Phase 的每个 Step 记录方法、结果、可追溯信息
**v2.0 规则**: 任何操作（运行/修改/测试/调试）必须在本文件记录，无一例外

---

## Worklog 格式规范

每个 Step 的记录应包含以下字段：

```
### Step XXa: [脚本名称]
- **方法**: 算法/工具/参数描述
- **输入**: 文件路径（大小/行数）
- **输出**: 文件路径（大小/行数）
- **关键结果**: 核心数字/结论
- **代码位置**: scripts/XX_phase/XXa_xxx.py:行号
- **SPEC 关联**: G{n} → ADR-{m}
- **OODA 状态**: Observe | Orient | Decide | Act
- **文献验证**: PMID:XXXXX 或 DOI 或 "待验证"
- **输出 README**: [如产生新输出目录，注明 README_文件说明.md 已创建/更新]
- **下一步**: 描述
```

### OODA 状态说明

| 状态 | 含义 | 必须记录的内容 |
|------|------|---------------|
| **Observe** | 执行分析、加载数据 | 脚本路径、输入文件、预期目标、执行结果 |
| **Orient** | 与文献对比、评估合理性 | 文献来源(PMID)、对比结论、意外发现 |
| **Decide** | 基于证据决定下一步 | 决策内容、理由、研究者确认状态 |
| **Act** | 执行决策 | 执行内容、变更范围、新增输出 |

### 强制记录场景（W1 规则检查清单）

以下操作**必须**在执行前/后记录到 WORKLOG：

- [ ] 运行任何分析脚本
- [ ] 修改现有脚本（参数调整、逻辑修改）
- [ ] 新增分析脚本
- [ ] 数据格式转换或预处理
- [ ] 生成或修改可视化
- [ ] 安装依赖或配置环境
- [ ] 修复 Bug
- [ ] 与研究者讨论后调整方案
- [ ] Gate Check 结果
